## 1. 프로젝트 개요

### 1.1 목표
- Claude Code의 핵심 CLI 기능을 gemma4 (Ollama 로컬 모델)로 대체 구현
- 파일/폴더 직접 접근, 이미지 분석, 코드 실행, 대화 히스토리 등 지원
- 완전 로컬 동작 — 인터넷 연결 불필요, API 비용 없음
- Ubuntu 22.04 + RTX 5070 + CUDA 12.8 환경 최적화

### 1.2 기술 스택
```
언어:        Python 3.11+
AI 백엔드:   Ollama (gemma4:e4b / gemma4:26b)
의존성:      ollama, rich, prompt_toolkit, watchdog
설치:        pip install ollama rich prompt_toolkit watchdog
실행:        python3 gemma-cli.py [옵션]
```

### 1.3 프로젝트 구조
```
gemma-cli/
├── gemma-cli.py          # 메인 진입점
├── core/
│   ├── __init__.py
│   ├── session.py        # 대화 세션 관리
│   ├── file_handler.py   # 파일/폴더 처리
│   ├── image_handler.py  # 이미지 처리
│   ├── code_runner.py    # 코드 실행
│   ├── git_handler.py    # Git 연동
│   └── ollama_client.py  # Ollama API 클라이언트
├── commands/

├── config/

│   └── profiles.py       # 프로젝트 프로파일
├── utils/
│   ├── syntax.py         # 코드 하이라이팅
│   └── token_counter.py  # 토큰 계산
└── tests/
    └── test_*.py
```

### 2.1 파일 시스템 접근

#### ✅ F-001: 단일 파일 읽기
```
트리거:  @파일경로
예시:    @main.c 이 파일 분석해줘
동작:
  - UTF-8 / EUC-KR 인코딩 자동 감지
  - 파일 크기 경고 (>500KB)
  - 언어 자동 감지 → 코드블록 래핑
  - 파일 메타정보 표시 (크기, 수정일, 라인수)
```

#### ✅ F-002: 다중 파일 읽기
```
트리거:  @파일1 @파일2 @파일3
예시:    @main.c @ocpp.c @uart.h 전부 분석해줘
동작:
  - 파일별 구분자로 컨텍스트 구성
  - 전체 토큰 수 계산 및 경고
  - 최대 20개 파일 동시 처리
```

#### ✅ F-003: Glob 패턴 파일 읽기
```
트리거:  @패턴
예시:    @src/**/*.c 전체 C 파일 분석
예시:    @include/*.h 헤더 파일 목록화
동작:
  - 재귀 glob 지원 (**)
  - 바이너리 파일 자동 제외
  - 파일 수 초과 시 (>30개) 확인 요청
  - 결과 파일 목록 미리보기 제공
```

#### ✅ F-004: 폴더 전체 읽기
```
트리거:  @폴더경로
예시:    @~/esp32-project 프로젝트 구조 분석해줘
동작:
  - .gitignore 규칙 적용하여 제외
  - 소스파일 우선 수집 (.c .h .py .js .ts .md)
  - 폴더 트리 구조 먼저 출력
  - 파일 내용 순차 포함
  - node_modules, .git, build 자동 제외
```

#### ✅ F-005: 파일 자동 감시 (Watch 모드)
```
트리거:  /watch @파일경로
예시:    /watch @main.c
동작:
  - 파일 변경 시 자동으로 컨텍스트 업데이트
  - "파일이 변경됐습니다. 다시 분석할까요?" 알림
  - watchdog 라이브러리 사용
```

#### ✅ F-006: 파일 쓰기 / 수정
```
트리거:  AI 응답에서 코드블록 감지 시
동작:
  - "이 코드를 파일에 저장할까요? [y/N]" 확인
  - 저장 전 diff 미리보기 표시
  - 백업 파일 자동 생성 (.bak)
  - 저장 후 git add 여부 확인
```

### 2.3 코드 실행

#### ✅ C-001: 인라인 코드 실행
```
트리거:  /run @파일경로
예시:    /run @test.py
지원:    Python, Bash, C (gcc), JavaScript (node)
동작:
  - 샌드박스 환경에서 실행
  - stdout / stderr 캡처하여 대화에 포함
  - 실행 시간 측정
  - 타임아웃 설정 (기본 30초)
```

#### ✅ C-002: AI 생성 코드 즉시 실행
```
동작:
  - AI가 코드 생성 시 "실행할까요? [y/N]" 제안
  - 임시 파일 생성 후 실행
  - 실행 결과를 다음 대화 컨텍스트에 포함
  - "실행 결과를 바탕으로 수정해줘" 가능
```

#### ✅ C-003: 셸 명령어 실행
```
트리거:  !명령어
예시:    !ls -la src/
예시:    !ninja -C build
동작:
  - 현재 디렉터리에서 셸 실행
  - 결과 대화에 포함
  - 위험 명령어 경고 (rm -rf 등)
```

### 2.4 Git 연동

#### ✅ G-001: Git 상태 자동 포함
```
동작:
  - 대화 시작 시 git status 자동 실행
  - 변경된 파일 목록을 컨텍스트에 포함
  - "변경된 파일 3개 감지됨" 표시
```

#### ✅ G-002: Diff 분석
```
트리거:  /diff 또는 /diff @파일
동작:
  - git diff 결과를 AI에게 전달
  - "이 변경사항 리뷰해줘" 자동 프롬프트
```

#### ✅ G-003: 자동 커밋 메시지 생성
```
트리거:  /commit
동작:
  - git diff --staged 내용 분석
  - Conventional Commits 형식으로 메시지 생성
  - 확인 후 git commit 실행
```

#### ✅ G-004: 브랜치 정보 표시
```
동작:
  - 프롬프트에 현재 브랜치명 표시
  - [main] [나] > 형식
```

### 2.5 대화 관리

#### ✅ D-001: 대화 히스토리 유지
```
동작:
  - 세션 내 전체 대화 유지
  - 토큰 수 실시간 표시
  - 컨텍스트 한도 초과 시 오래된 메시지 자동 압축
  - 시스템 프롬프트는 항상 유지
```

#### ✅ D-002: 대화 저장 / 불러오기
```
트리거:  /save [이름], /load [이름]
저장위치: ~/.gemma-cli/sessions/
형식:    JSON (timestamp, model, messages)
동작:
  - /save ocpp-debug → 현재 세션 저장
  - /load ocpp-debug → 이전 세션 복원
  - /sessions → 저장된 세션 목록
```

#### ✅ D-003: 대화 초기화
```
트리거:  /clear
동작:
  - 대화 히스토리 초기화
  - 시스템 프롬프트는 유지
  - "대화가 초기화됐습니다." 확인 메시지
```

#### ✅ D-004: 메시지 재시도
```
트리거:  /retry
동작:
  - 마지막 AI 응답 재생성
  - temperature 살짝 조정하여 다양한 답변
```

#### ✅ D-005: 대화 압축 (요약)
```
트리거:  /compress
동작:
  - 이전 대화를 AI가 요약
  - 요약본으로 히스토리 교체
  - 토큰 절약
```
